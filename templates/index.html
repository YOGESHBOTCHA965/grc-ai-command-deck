GRC AI System Operations Manual & Guide

Documentation detailing the architecture, phases, and operations of the GRC Compliance Engine.

Overview

The GRC AI Command Deck is a production-grade governance and compliance monitoring suite. It utilizes machine learning to ingest cloud provider log streams, isolate anomalous actions (Policy Drift), and map those events to specific NIST SP 800-53 Rev 5 control regulations using Semantic BERT mappings.

The 5-Phase Pipeline Architecture

1
NIST Control Extraction

Scrapes/loads controls catalog (JSON/CSV) from standard NIST repositories. Extracts control codes, title headers, and statement descriptions, cataloging them by security families.

2
Log Synthesis / Ingestion

Simulates CloudTrail-style logs capturing critical access actions, API events, IAM states, region settings, and MFA usage flags.

3
Unsupervised Anomaly Isolation

Trains a Scikit-Learn IsolationForest on categorical properties (user roles, API verbs, source IPs, MFA status). Anomalous logs with high outliers score are flagged as Compliance Drift.

4
SBERT Semantic Matching (NIST & MITRE)

Leverages SBERT (SentenceTransformer) to match the descriptive text of each flagged log to all 1,200+ NIST controls AND a curated catalog of MITRE ATT&CK Enterprise Cloud techniques, calculating similarity scores for both frameworks.

5
Human-in-the-Loop (HITL) & Report Generation

If SBERT mapping confidence falls below the similarity threshold for either NIST or MITRE, the log event is routed to the HITL dashboard. Reviewers resolve mapping states directly, compiling audit-ready PDF reports.

Dashboard Operational Guide

01

Authenticate Session

Log in using one of the pre-configured role-based credentials. Different roles afford access privileges (Admins/Analysts run pipeline, Reviewers audit).

02

Trigger Pipeline Run

Go to Overview, configure log count and SBERT similarity cutoff, and click Run Full Pipeline. A background worker thread handles execution while keeping the interface responsive.

03

Resolve Mappings (HITL)

Review flagged drift events on the Mappings & HITL page. For low-confidence mappings, click the entry, select suggestions, or insert custom Control IDs, then submit validation decisions.

04

Audit Report Rendering

Inspect the markdown report on the Compliance Report page. Note how verified and adjusted human override decisions are immediately compiled in the report statement.

05

Export Evidence Bundles

Export the formatted PDF document or download the CSV ZIP bundle. File bundles include the raw logs, scored anomalies, mapping logs, and the markdown reports.

06

Inspect Job History

Browse detailed execution audits in the Run History panel. Inspect specific timestamps, user triggers, parameters, errors, and performance details.

Core Optimization: Smart Caching

To solve the latency challenges of running neural networks on CPU, this app implements a dual-layer caching strategy:

• Model Caching: The SentenceTransformer PyTorch model is kept warm in memory, avoiding disk reads on every run.
• NIST Controls Embeddings Caching: All 1,200+ control statement embeddings are pre-calculated and cached to memory and a serialized pickle file (data/cache/).
• FastAPI Startup Warming: On server launch, a background daemon thread performs warming, loading the SBERT model and compiling embeddings. This means the very first execution runs instantly.
